Enunciado - Práctico Integrador 2026
Descripción General Como práctico integrador se solicita la creación de un Sistema de Gestión de Eventos y Entradas (tipo Ticketek), donde se destacan dos componentes principales:
Backend: desarrollado en Golang, que brindará todas las interfaces (API REST) necesarias para dar solución a los requerimientos del negocio. Frontend: desarrollado en React, que implementará las vistas finales y consumirá los servicios provistos por el backend.
1. Control de Versiones
El código fuente debe ser manejado como repositorio y debe estar subido y disponible en Github. El desarrollo debe ser colaborativo y comprobable entre todos los 3 o 4 integrantes del grupo. Debe emplearse el flujo de trabajo GitFlow (ramas main , develop y featurebranches ).
2. Base de Datos
Como fuente de datos se utilizará un esquema MySQL. El modelo de datos deberá soportar, como mínimo, tres entidades principales:
Usuarios Eventos Entradas El equipo tiene libertad para agregar entidades adicionales según las necesidades del dominio. Se pueden agregar entidades secundarias o intermedias si el diseńo lo requiere. Las tablas deben crearse y mapearse estrictamente utilizando el ORM GORM, implementando correctamente: claves foráneas relaciones correspondientes en las estructuras de Go

3. Backend
El backend debe implementar una arquitectura MVC (Modelo-Vista-Controlador).
Autenticación de usuarios Desarrollar endpoints que permitan implementar un sistema de login que maneje la sesión del usuario mediante JWT.
Requisitos
Soporte para dos tipos de usuarios: Cliente Administrador
El token debe expirar luego de un tiempo definido.
Funcionalidades para Cliente
A. Exploración del Catálogo de Eventos
Descripción Permitir visualizar el listado general de eventos disponibles. Implementar al menos un mecanismo de filtro.
Permisos No requiere validación de token.
B. Detalle Específico de un Evento
Descripción Obtener información completa de un evento específico mediante su ID.
Permisos No requiere validación de token.
C. Compra de Entrada
Descripción Procesar la adquisición de entradas. Contemplar validaciones de negocio.

Permisos Requiere validación de token.
D. Historial de Entradas ("Mis Entradas")
Descripción Retornar únicamente las entradas del usuario autenticado.
Permisos Requiere validación de token.
E. Cancelación de Compra
Descripción Procesar la anulación de tickets previamente adquiridos. Definir qué sucede con el cupo liberado.
Permisos Requiere validación de token.
F. Traspaso de Entrada
Descripción Transferir una entrada a otro usuario del sistema.
Permisos Requiere validación de token.
Funcionalidades para Administrador
A. Creación de Evento
Descripción Registrar nuevos eventos. Aplicar validaciones de consistencia.
Permisos Requiere validación de token.

B. Actualización de Evento
Descripción Modificar atributos de eventos existentes.
Permisos Requiere validación de token.
C. Cancelación de Evento
Descripción Eliminar eventos del catálogo.
Permisos Requiere validación de token.
D. Reportes y Métricas
Descripción Obtener métricas de ocupación o ventas. Ejemplos: entradas vendidas vs capacidad total listado de compradores
Permisos Requiere validación de token.
Seguridad Las operaciones sensibles deben protegerse con JWT firmado. Las contraseńas no deben almacenarse en texto plano. Debe utilizarse hashing: MD5 SHA256 u otro mecanismo equivalente
